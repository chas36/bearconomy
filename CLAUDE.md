# CLAUDE.md — Проект «Демидов» (рабочее название)

Экономический симулятор с сюжетом. Сеттинг: развитие частного капитала в
послепетровской России (~1725–1800). Референсы: Patrician 3, Port Royale 3,
The Guild 2, Plutocracy, Tropico. Соло-разработчик + AI-агенты.

## Технологии

- **Движок:** Godot 4.x (актуальный стабильный), язык — **GDScript**.
- Игра **data-first**: игрок живёт в интерфейсах, картах и событиях.
  3D/графика вторична и откладывается. Не предлагать миграцию на UE/Unity.
- Ядро симуляции — чистая логика, отделённая от сцен и UI (см. Архитектура).

## Команды

```bash
# Headless-прогон симуляции (основной цикл разработки ядра)
godot --headless --script econ_core.gd

# Запуск игры/сцены (когда появится UI)
godot --path .

# Тесты (GUT, когда подключим)
godot --headless -s addons/gut/gut_cmdln.gd

# Линт/формат вручную (на коммите запускается git-хуком сам)
pre-commit run --all-files
```

## Архитектура

```
econ_core.gd — headless-точка входа: демо-сценарий, псевдо-логистика, печать тиков
/sim        — чистая симуляция, НЕ знает о Node/Control/сценах
  goods.gd        — товары, базовые цены
  labor.gd        — типы труда (наёмные/приписные/посессионные)
  recipes.gd      — производственные цепочки
  trade_node.gd   — узел (город/ярмарка): склад, спрос, сглаженные цены
  enterprise.gd   — предприятия
  economy.gd      — тик экономики (порядок фаз строго фиксирован), buy/sell
/game       — связка симуляции с Godot (автолоады, сохранения) — позже
/ui         — сцены интерфейсов (Control-ноды) — позже
/data       — баланс в текстовых ресурсах (JSON/tres), НЕ хардкод — позже
/tests      — юнит-тесты симуляции — позже
```

Файлы `/sim` связаны через `preload`, наследуют `RefCounted`.

Правило: `/sim` должен запускаться и тестироваться headless без единой
сцены. UI только читает состояние симуляции и шлёт команды.

## Ключевые проектные решения (НЕ пересматривать без явного запроса)

1. **Экономика детерминированная.** Спрос/предложение/цены — явная
   математика, никаких нейросетей внутри симуляции. LLM используется
   только для нарратива/диалогов NPC (отдельный слой, позже).
2. **Цены локальные по узлам**, не глобальный рынок. Целевая цена v0:
   `base * clamp(target_stock / max(stock, 0.1), 0.25, 4.0)`.
   Текущая цена — состояние узла, движется к целевой на 20% за тик
   (`TradeNode.PRICE_SMOOTHING`); все сделки идут по текущей.
3. **Три типа труда с разными свойствами** — ядро уникальности игры:
   - Наёмные: платятся деньгами, эффективность 1.0, мобильны.
   - Приписные: даются государством, дёшевы, кормятся зерном, эфф. ~0.7.
   - Посессионные: покупаются вместе с заводом, неотчуждаемы отдельно.
   Труд — всегда явный вход производства, не абстрактная «стоимость».
4. **Порядок фаз тика:** производство → потребление → сглаживание цен
   → действия игрока/ИИ. Не менять порядок молча.
5. Первая цепочка: **руда → чугун → железо** (демидовская линия).
   Вторая по плану: зерно → мука/водка (винные откупа).

## Стиль кода

- GDScript со статической типизацией (`var x: float`, `-> void`).
- Идентификаторы и код — на английском; строки для игрока, комментарии
  и печать — на русском.
- Баланс (цифры) выносить в константы/data-файлы, не размазывать по логике.
- Малые функции, без преждевременных абстракций: v0-заглушки помечать
  комментарием `# v0-заглушка: ...`.

## Рабочий процесс

- Изменил `/sim` → прогнал headless-скрипт → показал вывод тиков в ответе.
- Изменения баланса подтверждать прогоном: цены не должны прилипать к
  клампам (0.25х/4х) на стабильной экономике — это признак разбаланса.
- Коммиты мелкие, сообщения на русском, формат: `sim: ...`, `ui: ...`,
  `data: ...`, `docs: ...`, `chore: ...`.
- На коммите git-хуком идёт pre-commit (гигиена + gdlint/gdformat).
  Если хук переформатировал файлы — `git add` и повторить коммит;
  `--no-verify` не использовать.
- Ничего не удалять и не переименовывать массово без явного запроса.

## Дорожная карта (ориентир, сверяться при выборе задач)

1. ✅ Скелет ядра: 3 узла, 1 цепочка, 3 типа труда, псевдо-торговля.
2. ✅ Сглаживание цен + базовый баланс. Известный хвост: рудник
   перепроизводит руду, вдолгую её цена уползёт к нижнему клампу —
   лечится рынком труда (п. 4) или поджатием мощности в сценарии.
3. Караваны: маршруты с временем в пути вместо «телепорта» товара.
4. Рынок труда: наём/отток от зарплаты, приписные через отношения с казной.
5. Минимальный UI: панель узла (склад/цены), панель предприятия, кнопки
   купли/продажи, спидометр тиков.
6. Сохранения (сериализация состояния /sim в JSON).
7. Событийная система (текстовые события с выбором) → потом слой LLM-NPC.

## Исторический контекст (для событий, названий, текстов)

- Эпоха: Анна Иоанновна → Елизавета → Екатерина II; дворцовые перевороты.
- Опорные реалии: Демидовы и уральские заводы, Макарьевская ярмарка,
  волжская логистика (бурлаки, Вышневолоцкая система), винные и соляные
  откупа, Табель о рангах, приписные/посессионные крестьяне.
- Тон текстов: живой русский язык, без канцелярита и без клюквы.

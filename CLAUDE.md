# CLAUDE.md — Проект «Демидов» (рабочее название)

Экономический симулятор с сюжетом: становление частного капитала в
петровской и раннепослепетровской России (~1700–1745), по демидовской
дуге от тульского оружейного дела к уральским заводам. Игрок вдохновлён
домом Антуфьевых-Демидовых, но не обязан повторять биографии Никиты или
Акинфия. Референсы: Patrician, Port Royale, The Guild, Plutocracy,
Transport Fever, Railway Empire, Tropico, City of Gangsters,
Cities: Skylines, Chocolatier / «Шокалатор» (серия).
Соло-разработчик + AI-агенты.

- **Движок:** Godot 4.x, язык — **GDScript**. Миграцию на UE/Unity не предлагать.
- Игра **data-first**: игрок живёт в интерфейсах, картах и событиях;
  3D/графика вторична и откладывается.

## Команды

`godot` на этой машине не в PATH — запускать бинарь из .app:

```bash
GODOT=/Applications/Godot.app/Contents/MacOS/Godot

$GODOT --headless --script econ_core.gd     # основной цикл разработки ядра
$GODOT --path .                             # запуск игры с UI
$GODOT --headless --path . --quit-after 1   # smoke-загрузка main-сцены
$GODOT --headless --script llm_smoke.gd     # smoke-test LLM-клиента
pre-commit run --all-files                  # линт/формат (на коммите — хуком)
```

## Архитектура

```text
econ_core.gd — headless-точка входа: демо-сценарий, печать тиков
/sim        — чистая симуляция, НЕ знает о Node/Control/сценах
  goods.gd        — товары, базовые цены
  labor.gd        — типы труда (наёмные/приписные/посессионные)
  recipes.gd      — производственные цепочки
  trade_node.gd   — узел (город/ярмарка): склад, спрос, сглаженные цены
  caravan.gd      — груз в пути между узлами
  enterprise.gd   — предприятия
  economy.gd      — тик экономики (порядок фаз строго фиксирован), торговля, труд
  event_catalog.gd — data-first каталог событий
  gameplay.gd     — игровой слой: контракт, события, сейвы, LLM snapshot
  demo_scenario.gd — стартовый сценарий и авто-логистика демо
/game       — недетерминированные интеграции (env_loader, openrouter_npc)
/ui         — игровой экран (Control-ноды, всё строится кодом)
  main.gd + main.tscn — сборка экрана: планка, карта, вкладки, события
  ui_theme.gd     — палитра, шрифт Old Standard TT (OFL), иконки кодом
  game_text.gd    — тик -> дата (неделя/месяц/год), деньги, время в пути
  map_view.gd     — карта путей: узлы, дороги, обозы; клик по узлу
  top_bar.gd      — дата, деньги, связи с казной, скорость, сейвы
  city_panel.gd   — рынок узла, торг, снаряжение обозов, стройка
  works_panel.gd  — предприятия: труд, ставка, приписные, расширение
  contracts_panel.gd — приказная доска и взятые подряды
  journal_panel.gd   — обозы в пути и записи приказчика
  event_dialog.gd    — событие-письмо на пергаменте (LLM-пересказ)
/docs       — дизайн-доки и справочники (см. «Документы»)
/data       — баланс в текстовых ресурсах — позже
/tests      — тесты симуляции — появятся в M1 этапа v1
```

Файлы `/sim` связаны через `preload`, наследуют `RefCounted`.
Правило: `/sim` запускается и тестируется headless без единой сцены.
UI только читает состояние симуляции и шлёт команды.

## Ключевые проектные решения (НЕ пересматривать без явного запроса)

1. **Экономика детерминированная.** Спрос/предложение/цены — явная
   математика, никаких нейросетей внутри `/sim`. LLM может быть внешним
   честным конкурентом, генератором кандидатов событий или narrative renderer,
   но действует только через выданный snapshot и публичные API
   (см. `docs/llm-contract.md`).
2. **Цены локальные по узлам**, не глобальный рынок. Целевая цена:
   `base * clamp(target_stock / max(stock, 0.1), 0.25, 4.0)`.
   Текущая цена — состояние узла, движется к целевой на 20% за тик
   (`TradeNode.PRICE_SMOOTHING`); все сделки идут по текущей.
3. **Три типа труда с разными свойствами** — ядро уникальности игры:
   наёмные (деньги, эфф. 1.0, мобильны), приписные (от государства, дёшевы,
   кормятся зерном, эфф. ~0.7), посессионные (покупаются вместе с заводом,
   неотчуждаемы). Труд — всегда явный вход производства.
4. **Порядок фаз тика:** производство → потребление → сглаживание цен →
   прибытие караванов → рынок труда → действия игрока/ИИ.
   Не менять порядок молча.
5. Первая цепочка: **руда → чугун → железо** (демидовская линия).
   Вторая: зерно → мука/водка (этап v1, M3).

## Референсы и тон

- Подробная расшифровка игровых референсов: `docs/game-references.md`.
- Тон — живой и приключенческий, но без потери исторической плотности:

  расширение сети, а не только таблицу производства.

## Стиль кода

- GDScript со статической типизацией (`var x: float`, `-> void`).
- Идентификаторы и код — на английском; строки для игрока, комментарии
  и печать — на русском.
- Баланс (цифры) — в константы/data-файлы, не размазывать по логике.
- Малые функции, без преждевременных абстракций; заглушки помечать
  комментарием `# v0-заглушка: ...`.

## Рабочий процесс

- Изменил `/sim` → прогнал headless-скрипт → показал вывод тиков в ответе.
- Изменения баланса подтверждать прогоном: цены не должны прилипать к
  клампам (0.25х/4х) на стабильной экономике — признак разбаланса.
- Коммиты мелкие, сообщения на русском: `sim: ...`, `ui: ...`, `data: ...`,
  `docs: ...`, `tests: ...`, `chore: ...`. Отдельный коммит на каждый
  завершённый шаг; чужие незакоммиченные изменения в коммит не включать.
- На коммите git-хуком идёт pre-commit (гигиена + gdlint/gdformat). Если хук
  переформатировал файлы — `git add` и повторить; `--no-verify` не использовать.
- Ничего не удалять и не переименовывать массово без явного запроса.

# Контракт LLM-слоя

LLM не является частью симуляции и не принимает экономических решений.
Игровой слой готовит для него только snapshot через `Gameplay.to_llm_context()`.
Текущий потребитель — `game/openrouter_npc.gd`, который возвращает текст
сцены поверх уже готового события. Если ключа, сети или ответа модели нет,
игра остаётся на статическом `body` из `event_catalog.gd`.

## Состав snapshot (`to_llm_context()`)

- `schema_version`, `purpose`, `language`, `style_constraints` — версия
  контракта и жёсткие ограничения: русский язык, исторический тон, никаких
  новых численных эффектов.
- `world` — эпоха и темы сцены.
- `state.tick`, `player_money`, `state_relations`, `contract` — короткое
  состояние игрока и текущего заказа. (В этапе v1, M4: секция `contract`
  заменяется на `contracts: {open, active}`.)
- `state.nodes` — по каждому узлу: склад, целевой склад, текущая/целевая цена,
  потребление и рыночное давление (`shortage`/`surplus`/`stable`).
- `state.enterprises` — предприятие, узел, рецепт, мощность, эффективная
  мощность, ставка и штат по типам труда.
- `state.caravans` — груз, маршрут, остаток пути, продажа по прибытии.
- `pending_event` — id, заголовок, тело, теги, участники, место, ставки сцены,
  `llm_context` и варианты выбора с текстом, кратким эффектом и подсказкой тона.
- `recent_notices` — последние игровые сообщения как память краткого горизонта.

## Ограничения

LLM-выход должен быть декоративным/нарративным: варианты выбора и эффекты
остаются в `event_catalog.gd` и применяются только `gameplay.gd`.

## Конфигурация

Локальная конфигурация читается из `.env` по образцу `.env.example`:

```bash
OPENROUTER_API_KEY=
OPENROUTER_MODEL=openai/gpt-oss-20b:free
```

Smoke-test клиента: `llm_smoke.gd` (см. команды в CLAUDE.md).

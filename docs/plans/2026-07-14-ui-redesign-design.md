# Дизайн: редизайн UI и графики под референсы

Дата: 2026-07-14. Статус: утверждён.

Цель — привести вид игры к уровню референсов (Patrician, Port Royale,
The Guild): живая карта, ощущение места и людей, оформленные экраны
вместо голых таблиц. Симуляция `/sim` не меняется; UI по-прежнему
только читает состояние и шлёт команды через публичные API.

Утверждённые боли текущего интерфейса: карта пустая и схематичная,
панели — голый текст, нет ощущения места и людей.

## Решения

- **Ассеты — внешняя AI-генерация (gpt-image / DALL-E).** Claude пишет
  полную спеку с промптами (`docs/asset-brief.md`), разработчик
  генерирует PNG и складывает в `ui/assets/gen/`. Промпты начинаются с
  общего «стилевого якоря»: русская гравюра/парсуна начала XVIII века,
  палитра «старой конторы».
- **Фолбэки обязательны.** Каждое место, ждущее PNG, проверяет наличие
  файла; при отсутствии — текущая процедурная отрисовка. Игра всегда
  запускается даже с пустой папкой ассетов.
- **Палитра и шрифты `ui_theme.gd` остаются базой** — общий
  арт-дирекшн не пересматриваем, наращиваем поверх.

## Композиция: два главных экрана

### Экран карты (по умолчанию)

- Карта на весь экран. Фон — один PNG ~2048×1152: старинная
  гравированная карта России (пергамент, рельеф штриховкой, леса,
  Волга/Кама/Чусовая, компас, картуш). География сразу под этап v2:
  Москва, Тула, Макарьево, Невьянск, Петербург; узлы, которых нет в
  симуляции, маркеров не получают.
- Привязка координат — data-файл раскладки (нормализованные 0..1
  координаты узлов); картинка масштабируется letterbox, перегенерация
  карты требует лишь поправить числа.
- Поверх кодом: маркеры городов (AI-виньетки в картуше + название,
  пергаментный тултип со сводкой цен), маршруты лёгкой дугой, обозы —
  AI-спрайт повозки ~48 px с флажком цвета владельца.
- Камера фиксированная, пан/зум не делаем (YAGNI при 3–5 узлах).
- Сверху — глобальная планка (дата, деньги, казна, скорость, сейвы,
  кнопка «Летопись»), снизу — строка приказчика.
- Клик по городу → городской экран.

### Городской экран

- Полноэкранная интерактивная сцена в духе Patrician: фон-панорама
  1920×1080 + 3–5 отдельных PNG-спрайтов зданий с прозрачным фоном.
- Здание — `TextureButton` с кликом по альфа-маске
  (`texture_click_mask`); позиции/масштабы — в data-файле раскладки.
- Наведение: подсветка модуляцией + золотая подпись названия здания.
- Здания по типу узла:
  - Невьянск: домна с дымом, заводская контора, торговые ряды,
    ямской двор;
  - Макарьево: ярмарочные ряды, контора, ямской двор;
  - Москва: торговые ряды, приказная изба (вход к казённым заказам),
    контора, ямской двор.
  Доска контрактов остаётся глобальной механикой — изба лишь вход;
  если по геймплею неудобно, избу добавим во все города.
- Клик по зданию открывает поверх сцены центрированную панель-«бумагу»
  (~700 px): пергаментная 9-patch-рамка, заголовок с печатью, внутри —
  переработанный код текущих панелей. Маппинг вкладок на здания:
  Торговые ряды → рынок/торг, Заводская контора → предприятия,
  Приказная изба → заказы, Ямской двор → снаряжение обозов.
- Esc/крестик закрывает бумагу; кнопка «На карту» возвращает на карту.
  Симуляция продолжает тикать на городском экране.

## Люди

- Пул портретов по ролям (подьячий, купец, приказчик, мастер,
  староста, офицер), по 2–3 варианта, ~512×512, парсуна/гравюра.
- События: `event_catalog.gd` получает поле `speaker_role` (данные);
  диалог письма показывает портрет, имя и роль слева от пергамента.
- Заказы: карточка контракта показывает «выдавшее лицо» — роль по типу
  заказчика (казённый → подьячий, частный → купец), вариант портрета и
  имя выбираются стабильным хешем от полей контракта.
- Детерминизм: никакого RNG — выбор портретов/имён только хешами,
  в сейв и в `/sim` ничего не добавляется, UI вычисляет на лету.
- Имена — data-список историчных имён эпохи в UI-слое
  (`ui/persona.gd`).

## Ассеты

Структура: `ui/assets/gen/{map,cities,buildings,goods,portraits,chrome}/`.

Состав (~40 файлов): 1 карта-фон, 5 виньеток-маркеров узлов,
3 панорамы городов, ~10 спрайтов зданий, 6 иконок товаров (гравюрный
стиль), 12–15 портретов, пергаментная рамка-9patch, печать, спрайт
обоза. Точные имена, размеры и промпты — в `docs/asset-brief.md`.

## Порядок работ

1. `docs/asset-brief.md` + data-файл раскладки — генерация ассетов
   идёт параллельно разработке.
2. Перестройка кода на фолбэках: экран карты, городской экран,
   панели-бумаги, портреты в событиях/заказах.
3. Интеграция ассетов по мере готовности, полировка по скриншотам.

## Проверка

- Скриншот-скрипт в репо (`tests/screenshot.gd`) — каждый шаг
  подтверждается картинкой.
- Headless-регрессии `/sim` не трогаются и должны проходить.
- Обновление CLAUDE.md (структура `/ui`) — в том же коммите, что и
  перестройка.

# Демидов (рабочее название)

Экономический симулятор с сюжетом о становлении частного капитала в
петровской и раннепослепетровской России (~1700–1745): тульское оружейное
дело, уральские заводы, ярмарки, волжская логистика, казённые заказы.
Игрок вдохновлён домом Антуфьевых-Демидовых, но не обязан повторять
реальные биографии Никиты или Акинфия. Референсы: Patrician 3,
Port Royale 3, The Guild 2.

Первая производственная цепочка — демидовская: **руда → чугун → железо**.
Ядро уникальности — три типа труда с разными свойствами: наёмные,
приписные и посессионные крестьяне.

## Технологии

- [Godot 4.x](https://godotengine.org/), GDScript.
- Игра data-first: интерфейсы, карты и события; 3D вторично.
- Ядро симуляции (`/sim`) — чистая детерминированная логика без сцен и UI.

## Запуск

```bash
# Headless-прогон симуляции (основной цикл разработки ядра)
godot --headless --script econ_core.gd

# Запуск игры/сцены (когда появится UI)
godot --path .

# Smoke-test временного LLM-NPC слоя через OpenRouter
godot --headless --script llm_smoke.gd
```

## Структура

```
econ_core.gd — headless-точка входа: демо-сценарий и печать тиков
/sim         — чистая симуляция (товары, труд, рецепты, узлы, предприятия, тик)
/game        — недетерминированные интеграции вокруг симуляции
/ui          — сцены интерфейсов (позже)
/data        — баланс в текстовых ресурсах (позже)
/tests       — юнит-тесты симуляции (позже)
```

## Статус

Готов скелет ядра: 3 узла, цепочка руда→чугун→железо, 3 типа труда,
сглаживание локальных цен, караваны с временем в пути, базовый рынок труда
и минимальный UI. Добавлен игровой слой v0: московский заказ с дедлайном,
события с выбором, сохранение/загрузка JSON и data-first каталог событий.

Для текущего LLM renderer есть `Gameplay.to_llm_context()`: он отдаёт
состояние игрока, узлов, предприятий, караванов, текущего события, вариантов
выбора и нарративные ограничения. Стратегическое направление LLM-слоя —
честный конкурент, который видит ту же информацию, что и игрок, и действует
только через публичные игровые API; также возможна генерация кандидатов
событий для будущей базы. Временный OpenRouter-клиент лежит в `/game` и
читает локальный `.env` по образцу `.env.example`. Подробности — в
[CLAUDE.md](CLAUDE.md) и [docs/llm-contract.md](docs/llm-contract.md).
